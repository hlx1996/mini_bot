# mini_bot — multi-platform bot bridge (WeChat + Lark) powered by qoder

A clean ~2400-line Bash bot that bridges **WeChat** and **Lark/Feishu** to a
local **qoder** LLM. Designed for the case where you can't install heavyweight
agent frameworks on your work laptop, but still want OpenClaw-style features:
multi-turn memory, personas, RAG, sub-agents, scheduled tasks, web dashboard,
auto-routing, image generation, TTS, hooks, MCP — all out of one shell script
+ a small Python web panel.

> 中文文档请见 [README.md](./README.md)（更完整 / 持续更新）

## Quick start

```bash
git clone git@github.com:hlx1996/mini_bot.git
cd mini_bot

# 1. configure accounts
cat > state/accounts.list <<EOF
wechat:default
lark:bot   cat   lite
EOF

# 2. login (each transport has its own CLI)
python3 wxlink.py --account default login
lark-cli auth login --as bot

# 3. run
bash bot.sh run                     # bot loop
python3 web.py --host 0.0.0.0 --port 8787   # dashboard (optional)
```

Or with Docker:

```bash
docker compose up -d
```

## Architecture

```
WeChat user ──→ iLink ──→ WeixinClawBot ──→ wxlink.py subscribe ─┐
                                                                 ├─→ bot.sh
Lark user   ──→ Feishu API ──→ lark-cli event +subscribe ────────┘  (per-event handler)
                                                                          │
                                                                          ▼
                                                          qodercli + souls/memory/RAG/hooks
                                                                          │
                                                                          ▼
                                                       wxlink send-text/media · lark-cli api reply
```

One process hosts any number of WeChat numbers + Lark bots. Each is keyed by
`<platform>:<account>:<peer>` so sessions never collide.

## Web panel auth

Three modes; the first one that's configured wins:

| Env vars set | Mode |
|---|---|
| `MINIBOT_GH_CLIENT_ID` + `MINIBOT_GH_CLIENT_SECRET` (+ optional `MINIBOT_GH_ALLOWED_USERS=foo,bar`) | **GitHub OAuth** — `/login` redirects to GitHub; cookie session |
| `MINIBOT_USER` + `MINIBOT_PASS` | **HTTP Basic Auth** |
| *(none)* | **Open** (loopback only is recommended) |

GitHub OAuth setup:
1. Create an OAuth App at <https://github.com/settings/developers> with callback `http(s)://<your-host>/oauth/callback`.
2. Export the three env vars (or put them in docker-compose `environment:`).
3. Restart `web.py`.
